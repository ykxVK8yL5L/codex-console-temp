# v1.1.2

发布日期：2026-03-31

## 本次新增

- 新增统一鉴权与安全基线，补齐 `src/web/auth.py`、首次改密页面、API 与 WebSocket 鉴权统一入口。
- 新增系统自检能力、自检页面、自检调度器和修复中心。
- 新增统一任务中心，统一管理注册、支付、自检等任务状态。
- 新增周期任务调度，支持计划任务创建、编辑、启停、立即执行与轮询管理。
- 新增 New-API 服务配置与上传能力，支持单账号、批量与注册成功后自动上传。
- 新增卡池功能、自动绑卡能力与相关上游对接能力。
- 新增 Alembic 数据库迁移体系。
- 新增多组测试与 CI 工作流。

## 本次优化

- 优化前端请求、轮询、去重、超时与并发控制。
- 优化自动注册补货与库存监控链路，增强任务取消感知、批次统计与监控刷新。
- 优化注册链路容错，补齐 continue URL、workspace、OTP 重试和多入口处理。
- 优化账号模型、支付历史快照、审计记录和配置能力。
- 优化数据库部署与运行配置，补齐 PostgreSQL 默认 compose、连接池调优参数和设置页数据库后端展示。
- 优化默认 Docker PostgreSQL 暴露方式，改为仅绑定 `127.0.0.1:5432`，便于本机迁移与宿主机直连，同时减少对外暴露面。
- 优化仓库忽略规则，补充忽略本地 PostgreSQL 数据目录 `pgdata/`，避免误提交运行态数据。
- 优化 Docker 构建上下文，新增 `.dockerignore` 并排除 `pgdata/`、`data/`、`logs/` 等本地运行目录，减少误打包与构建体积。
- 优化本地启动体验，新增 `start_webui.py` 启动器，自动拉起 PostgreSQL、等待健康检查并启动 `webui.py`。
- 优化默认数据保留策略：日志保留天数调整为 14 天，过期任务清理默认改为分层处理 completed/cancelled、failed、pending、running 四类记录。
- 补充 SQLite -> PostgreSQL 数据迁移脚本，便于把现有 data/database.db 导入新的 PostgreSQL 实例。
- 优化公告、Footer、外链跳转、页面样式与部分交互细节。

## 本次修复

- 修复自动补货首次执行后不再按间隔继续执行的问题。
- 修复自动补货批次取消时子任务未正确同步取消的问题。
- 修复并行模式与流水线模式下取消响应不及时、统计不准确的问题。
- 修复注册后自动上传链路对仅拿到 `access_token` 的账号缺少过滤与统计的问题；新增前端默认过滤开关，并在批量/自动注册收尾日志中输出 token 完整性统计。
- 修复监控页最近检查时间与库存未及时刷新的问题。
- 修复注册页刷新后自动恢复监控的异常行为。
- 修复首页今日统计中“注册”和成功率误按请求总数计算的问题，统一改为按成功数与失败数汇总。
- 修复 Web UI 启动日志把 `0.0.0.0` 当成访问地址展示的问题，改为优先提示本机可直接访问的 `127.0.0.1`。
- 修复 `start_webui.py` 通过子进程启动时 Windows 终端下 `Ctrl+C` 退出不稳定的问题，改为同进程启动 `webui.py`。
- 修复定时自动一键刷新链路不可用的问题。
- 修复公告按钮样式、Footer 外链与 Blog 跳转异常。
- 修复多邮箱服务接入、CloudMail / LuckMail / YYDS Mail 等兼容问题。
- 修复删除仍被注册任务引用的邮箱服务时触发数据库外键异常并返回 500 的问题，改为返回可读的冲突提示。
- 优化邮箱服务列表操作列，将启用 / 禁用从“更多”菜单前置为直接按钮，便于在删除受限时快速禁用服务。
